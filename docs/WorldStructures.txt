   WorldStructure JSON Documentation

   How It Works

   The engine provides a node editor that constructs biome configurations saved as JSON files. These are loaded by the asset system using WorldStructureAsset codec. The "Type" field specifies which implementation to use - "NoiseRange" maps to BasicWorldStructureAsset.

   JSON Structure

   Root Level Fields

   Field                       │ Type    │ Required │ Default │ Description                                             
   ----------------------------+---------+----------+---------+---------------------------------------------------------
   `Type`                      │ String  │ Yes      │ -       │ Always `"NoiseRange"` - maps to BasicWorldStructureAsset
   `Biomes`                    │ Array   │ Yes      │ -       │ Array of biome range definitions
   `Density`                   │ Object  │ Yes      │ -       │ Defines noise function for biome selection
   `DefaultBiome`              │ String  │ Yes      │ -       │ Fallback biome ID when no range matches
   `DefaultTransitionDistance` │ Integer │ Yes      │ 32      │ Smoothing radius between biomes (must be > 0)
   `MaxBiomeEdgeDistance`      │ Integer │ Yes      │ 0       │ Maximum distance to biome edge for edge effects (≥ 0)
   `ContentFields`             │ Array   │ No       │ []      │ Height reference definitions for biomes

   Biomes Array Elements

   Each biome entry is a BiomeRangeAsset:

   Field   │ Type   │ Required │ Default │ Description                         
   --------+--------+----------+---------+-------------------------------------
   `Biome` │ String │ Yes      │ -       │ Biome asset ID to reference
   `Min`   │ Double │ Yes      │ -1.0    │ Minimum density value for this biome
   `Max`   │ Double │ Yes      │ 1.0     │ Maximum density value for this biome

   What Min/Max Mean: The density field generates a noise value at each position (typically -1.0 to 1.0). Biomes are selected based on which range [Min, Max] the noise value falls into.

   Density Object

   Defines the noise function. Common types:

   Imported Density (Node Editor Export)

   json
     {
       "Type": "Imported",
       "Name": "Biome-Map"
     }

   References an exported noise node from the node editor.

   Constant Density

   json
     {
       "Type": "Constant",
       "Value": 0.5
     }

   Simplex Noise 2D

   json
     {
       "Type": "SimplexNoise2D",
       "Seed": 123,
       "Frequency": 0.01,
       "Amplitude": 1.0,
       "Octaves": 4,
       "Persistence": 0.5,
       "Lacunarity": 2.0
     }

   ContentFields Array

   Defines named height references used by biomes:

   BaseHeight ContentField

   json
     {
       "Type": "BaseHeight",
       "Name": "Base",
       "Y": 100.0
     }

   Field  │ Type   │ Required │ Default │ Description                     
   -------+--------+----------+---------+---------------------------------
   `Type` │ String │ Yes      │ -       │ `"BaseHeight"`
   `Name` │ String │ Yes      │ -       │ Reference name for biomes to use
   `Y`    │ Double │ No       │ 0.0     │ Height value

   Key Processing Flow

   1. Load: Asset system reads JSON, uses WorldStructureAsset.CODEC to parse
   2. Type Resolution: "Type": "NoiseRange" → BasicWorldStructureAsset
   3. BiomeMap Building: Creates SimpleBiomeMap with:
     •  Density noise function for sampling
     •  Biome ranges for mapping density → biome
     •  Default biome for fallback
     •  Transition radius for smoothing
   4. Runtime: For each world position, sample density, select biome by range
